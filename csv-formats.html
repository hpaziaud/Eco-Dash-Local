🎯 Formats Supportés

EcoDash Local peut traiter différents formats de fichiers CSV grâce à sa détection automatique des colonnes. Vous n'avez pas besoin de respecter un format exact !

🔍 Détection Automatique

Reconnaissance intelligente des colonnes basée sur les noms d'en-têtes

🔄 Conversion d'Unités

Support automatique des unités : KB, MB, GB, bytes

📝 Colonnes Flexibles

Accepte différents noms de colonnes et langues

🛡️ Données Manquantes

Génération automatique de valeurs par défaut

📋 Colonnes Requises

EcoDash Local détecte automatiquement ces types de données :

● URL/Page (Recommandé)

Noms d'en-têtes supportés : url, page, path, page_url

● Taille de Page (Recommandé)

Noms d'en-têtes supportés : size, weight, poids, page_size_kb, size_mb

Unités supportées : KB, MB, GB, bytes

● Nombre de Requêtes (Optionnel)

Noms d'en-têtes supportés : requests, req, requête, http_requests

● TTFB (Optionnel)

Noms d'en-têtes supportés : ttfb, first, response, ttfb_ms

● Temps de Chargement (Optionnel)

Noms d'en-têtes supportés : load, time, temps, duration, load_time_ms

📄 Exemples de Formats

Format Standard (Recommandé)

url,page_size_kb,requests,ttfb_ms,load_time_ms /home,1200,45,180,900 /about,800,30,150,700 /contact,950,35,200,850

Format avec Unités MB

Page URL,Size (MB),HTTP Requests,First Byte (ms),Load Duration (ms) "https://example.com/",1.2,45,180,900 "https://example.com/about",0.8,30,150,700

Format Minimal (2 colonnes)

path,weight /home,1200 /about,800 /contact,950

Format avec Noms Français

page,poids_kb,requêtes,ttfb_ms,temps_chargement /accueil,1200,45,180,900 /a-propos,800,30,150,700

Format avec Données Supplémentaires

url,page_size_kb,requests,ttfb_ms,load_time_ms,images_count,js_files,css_files /home,1200,45,180,900,8,3,2 /about,800,30,150,700,5,2,1

🔧 Fonctionnalités Avancées

Gestion des Guillemets

Support des valeurs entre guillemets pour gérer les virgules dans les données :

"Page URL","Size (MB)","Description" "https://example.com/","1.2","Page d'accueil avec images" "https://example.com/about","0.8","Page à propos, simple"

Conversion Automatique d'Unités

EcoDash convertit automatiquement :

  • MB → KB : 1.2 MB devient 1228.8 KB
  • GB → KB : 0.5 GB devient 524288 KB
  • Bytes → KB : 1024000 bytes devient 1000 KB

Génération de Données par Défaut

Si des colonnes sont manquantes, EcoDash génère des valeurs réalistes :

  • URL manquante : Génère "page-xxxxx"
  • Taille manquante : Valeur aléatoire entre 100-2100 KB
  • Requêtes manquantes : Valeur aléatoire entre 10-110
  • TTFB manquant : Valeur aléatoire entre 50-550 ms
  • Temps de chargement manquant : Valeur aléatoire entre 200-2200 ms

⚠️ Limitations

  • Fichiers CSV uniquement (pas de Excel, JSON, XML)
  • Encodage UTF-8 recommandé
  • Maximum 1000 lignes recommandé pour les performances
  • Première ligne doit contenir les en-têtes

💡 Conseils

  • Testez d'abord avec les fichiers de test inclus
  • Vérifiez l'encodage de votre fichier CSV (UTF-8)
  • Utilisez des noms d'en-têtes clairs pour une meilleure détection
  • Évitez les caractères spéciaux dans les noms de colonnes
  • Consultez la console (F12) en cas de problème
